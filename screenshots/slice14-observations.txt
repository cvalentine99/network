   - Cloud Services toggle: OFF by default

4. Loading state:
   - Not separately screenshotted because the tRPC query resolves near-instantly
     in the dev environment. The loading state is a brief skeleton shimmer
     that appears for <100ms. The component code includes a loading branch
     that renders a skeleton card (verified by code inspection).

5. Error state:
   - Not separately screenshotted because triggering a transport error requires
     the tRPC server to be down, which would prevent Puppeteer from loading
     the page. The error state is tested in slice14.test.ts via fixture
     validation. The component renders an ErrorState component with
     "Failed to load appliance configuration" message.
